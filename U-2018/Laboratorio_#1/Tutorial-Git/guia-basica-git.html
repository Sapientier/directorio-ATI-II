Guía Básica de Git

Para explicar de manera breve el propósito de esta guía, basta con tomar en cuenta el siguiente escenario:

"Maria, Ana y Víctor trabajan desde hace 5 años como desarrolladores en un macro-proyecto web. María vive en Valencia, Ana tuvo que mudarse con su familia a Mérida, y Víctor continúa viviendo en Caracas".

"Víctor representa al líder del equipo, y tiene como alternativa buscar a nuevos desarrolladores que vivan en las cercanías de Caracas, debido a que Ana no volverá por un tiempo a Caracas, y María presenta una infinidad de problemas con respecto al transporte; pero quién mejor que María, y quién mejor que Ana, que representan aquellas entidades que han contribuido con el desarrollo de dicho proyecto desde su origen, y conocen cada uno de sus requerimientos".

"Por lo tanto, Víctor se da cuenta de que lo ideal sería que existiera una manera de que cada uno de ellos pueda contribuir con el proyecto rompiendo con la limitación geográfica que se les presenta. En este punto, Víctor encuentra una solución, un Sistema de Control de Versiones, el cual le permite a cada uno de ellos contribuir en un proyecto, independientemente de su geolocalización, realizando una adecuada gestión de los conflictos que podrían surgir al momento de modificar partes del código en común, y llevar a cabo un seguimiento de las modificaciones realizadas a medida que se desarrolla dicho sistema, esa solución es Git".

Primeros pasos con Git

El primer paso que se debe hacer para comenzar a utilizar Git es crear una cuenta desde la página oficial de GitHub .

Una vez que nos encontremos en la interfaz de la página de inicio, procedemos a realizar el registro, rellenando los campos correspondientes al mismo.

Luego realizamos la verificación de la cuenta, para ello debemos resolver un puzzle, seleccionar el plan que utilizaremos, en este caso será el gratuito, y completar una breve encuesta de la utilidad que le daremos a nuestra cuenta de GitHub.

Creando un repositorio

Un repositorio es básicamente el directorio que albergará nuestro proyecto, existen varias formas de crear un repositorio, pero lo más tradicional es realizar dicha acción desde nuestro usuario a través de la página web.

Para crear nuestro primer repositorio se debe seleccionar la opción

Como este tutorial se está realizando para fines académicos, aprovecharé la oportunidad para crear un repositorio que contendrá los entregables de cada uno de los estudiantes inscritos en la materia; así como también, los archivos correspondientes a las guías generadas por el preparador de la materia para cada uno de los laboratorios que serán dictados a lo largo del semestre.

Instalación y configuración de Git

Bien, ahora que hemos construido nuestro repositorio, ha llegado la hora de usarlo. Existen dos enfoques: utilizar la GUI que provee GitHub a través de su página web, o usar la línea de comandos para conectarse al repositorio y comenzar a subir y sincronizar nuestro contenido.

Utilizaremos el segundo enfoque puesto que somos estudiantes de Computación, y tenemos que acostumbrarnos a darle uso a la línea de comandos. El comando para instalar Git es:

~$ sudo apt-get install git

En mi caso ya se encuentra instalado, pero la instalación es sencilla, debe ingresar los datos referentes a su cuenta. Una vez realizada la instalación, procederemos a revisar la versión que se dispone, el comando para ello es:

~$ git --version

Ahora configuraremos nuestra cuenta de Git con los siguientes comandos:

~$ git config --global user.name [username]; git config --global user.email [useremail@email.com]

Hasta este punto, ¿qué tenemos? Sabemos cómo instalar y configurar Git, tenemos el conocimiento base de cómo crear un repositorio. Tomando en cuenta los objetivos cumplidos, ¿qué nos faltaría para completar la meta del presente laboratorio? El elemento vital que no hemos cubierto hasta el momento es el directorio de trabajo, también conocido como workspace, que representa aquel directorio que será incorporado en el repositorio (en la mayoría de los casos el repositorio no es más que el workspace), con la finalidad de realizar modificaciones sobre el mismo desde una perspectiva colaborativa sin afectar los avances del resto de los integrantes del equipo.

Nota: estamos asumiendo que el repositorio ya existe, y el directorio que tenemos en nuestra máquina local lo queremos incorporar a este repositorio, es importante aclarar este punto, debido a que lo común es construir un workspace e inicializarlo en Git, para así crear automáticamente el repositorio sin la necesidad de subir nuestro workspace a otro repositorio.

Incorporando Workspace en Git

Como se indicó anteriormente, ya el repositorio existe, de hecho, se creó en la sección Crear un repositorio, y se definió con el nombre directorio-ATI-II. Por lo tanto, si queremos trabajar dentro de ese repositorio será necesario "clonarlo" para posteriormente utilizarlo.

El comando que implementaremos será:

~$ git clone [url]

Aquí podría surgir una duda, ¿cómo saber qué url tiene mi repositorio? Esa información la podemos obtener desde nuestra cuenta de GitHub.

Ahora sí será posible clonar el proyecto, ya que tenemos la url del repositorio, y lo haremos a través del comando:.

~$ git clone https://github.com/Vgpollster/directorio-ATI-II.git

Se genera un warning ya que por los momentos nuestro repositorio se encuentra vacío, pero más adelante veremos cómo agregar los archivos pertenecientes a este laboratorio.

Bien, si accedemos al directorio donde realizamos el comando, notaremos que se ha construido un directorio con el nombre del repositorio directorio-ATI-II. Es en ese directorio donde subiremos los entregables y los documentos correspondientes a cada uno de los laboratorios que serán dictados durante el transcurso del semestre.


En resumen, se podría decir que la principal ventaja de este comando es que básicamente se descarga el repositorio remoto, y se inicializa localmente en su máquina. ¿Qué quiere decir esto? Que si clonas un repositorio, ya puedes trabajar directamente con él de manera local, para luego subir los cambios realizados a ese repositorio remoto alojado en GitHub.

Agregar archivos a repositorio remoto

Si recapitulamos todas las acciones que hemos realizado, brevemente podríamos decir que:

  1. Creamos nuestra cuenta en GitHub
  2. Creamos un repositorio, cuyo propósito será subir los archivos relacionados con la materia
  3. Configuramos Git a través de la línea de comandos
  4. Realizamos la operación clone de dicho repositorio, y ahora tenemos una copia local en nuestra máquina
  5. ¿Qué sigue?

Bien, nos queda agregar nuestros archivos a dicho repositorio. Sin embargo, este paso es totalmente sencillo, primero debemos acceder al directorio (la copia local que hemos agregado en nuestra máquina), y crear un nuevo directorio "Laboratorios/U-2018", desde aquí añadiremos una nueva carpeta que hará referencia al laboratorio, por ejemplo "Laboratorio_#1", y consecutivamente, agregaremos en la misma los archivos correspondientes a dicho laboratorio. Estos archivos son cuatro, un pre-laboratorio, el cual contiene una serie de preguntas que cada alumno debe responder, la parte teórica, que básicamente son los fundamentos teóricos del laboratorio, el post-laboratorio, que representan las actividades o entregables que cada alumno debe realizar para obtener una calificación, y su guía práctica respectiva (lo que está leyendo en este momento).

Add y commit en Git

A pesar de que el paso anterior fue sencillo, todavía no hemos agregado los archivos al repositorio remoto, si toman los apuntes teóricos notarán que es necesario agregar estos archivos por medio del comando add, para luego confirmarlos mediante commit. De hecho, si nos ubicamos en el directorio local de nuestro repositorio, y utilizamos el comando git status, ocurrirá lo siguiente:

Efectivamente, nos encontramos en la rama master (la rama principal del proyecto), pero aún no hemos agregado esos archivos, por eso no presentan ningún seguimiento. Entonces lo siguiente que haremos será agregar dichos archivos por medio del comando git add . (utilizo el . como parámetro de git add para dar a entender que quiero que agregue todos los archivos que se encuentren sin seguimiento desde el directorio local), para luego volver a realizar el comando git status para ver lo que ocurre.

Por último, nos queda realizar commit con su respectivo mensaje, y una vez que hagamos esto, solo nos quedará sincronizar los cambios con el repositorio remoto.

Sincronizar cambios

Localmente sabemos que nuestros archivos ya se encuentran cargados, pero NO en el repositorio remoto, sino en nuestro directorio de trabajo local. Lo siguiente que realizaremos será la sincronización de nuestros cambios en el repositorio, para ello aplicaremos los comandos base: git pull & git push.

Nota: Al momento de realizar push se nos pedirán los datos referentes a nuestra cuenta, es sencillo, colocamos nuestro nombre de usuario y contraseña (de GitHub por supuesto).

Después de todos estos pasos, podemos corroborar desde nuestro usuario que efectivamente se ha agregado la información al repositorio

Conflictos

Lo último que nos queda por ver es qué ocurre si modifico una sección de un archivo en dos instancias diferentes, y este punto es algo totalmente importante; sin embargo, no es necesario abarcar ese tópico en estos momentos debido a que este primer laboratorio tiene un alcance general, a medida que se comiencen a subir códigos al repositorio y trabajar de manera colaborativa se agregarán nuevas guías que cubran esos aspectos.

Evaluación
